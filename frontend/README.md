# Saint Laurent Consulting (SLC) Frontend

## Project Overview

The Saint Laurent Consulting frontend is a state-of-the-art, premium single-page application (SPA) built with React 19 and Vite. The digital platform is designed to embody a high-end, authoritative "Investment Banker" aesthetic, conveying trust, sophistication, and professional excellence. 

This repository serves as the digital face of Saint Laurent Consulting, showcasing services (e.g., HR strategy, people management), company culture (Careers, Mission), and an integrated partner network.

---

## Technology Stack

* **Core:** React 19, JavaScript (ES6+)
* **Build Tool & Dev Server:** Vite
* **Styling:** Vanilla CSS & CSS Modules
* **Routing:** React Router DOM (v6)
* **Icons:** `react-icons` (Specifically Feather and Material sets)
* **Containerization:** Docker (Multi-stage build with Nginx)

## 📂 Project Structure

For developers jumping into the codebase, here is the high-level architecture of the `src` directory:

```text
src/
├── assets/          # Static assets (images, logos) not handled by the public folder
├── components/      # Reusable UI components (Navbar, Footer, CustomCursor, Cards)
├── pages/           # Route-level components (Home, About, Careers, ServiceDetail)

└── index.css        # Global styles, CSS variables (Design Tokens), and reset
```

## 🛠 Developer Guidelines & Architecture

### 1. Styling Strategy & CSS Variables
The project avoids utility frameworks (like Tailwind) in favor of semantic Vanilla CSS to maintain total control over the bespoke animations and micro-interactions.
* **Global Tokens:** All theme colors and typography are defined in `src/index.css` under the `:root` pseudo-class (e.g., `--bg-beige`, `--accent`, `--serif`).
* **Component Styles:** Each component and page has a dedicated stylesheet in the `src/styles/` directory (e.g., `Navbar.css`, `Careers.css`). Always import the specific stylesheet directly into the `.jsx` file.

### 2. Design Aesthetic (The "Investment Banker" Style)
When creating new pages or components, adhere to the following rules:
* **Typography:** Use `--serif` (*Playfair Display*) for main headings/brand names and `--sans` (*Inter*) for body text and navigation.
* **Color Rhythm:** Use an alternating section background rhythm of Pure White (`#ffffff`) and Beige (`var(--bg-beige)`).
* **Headings:** Apply the `.section-title` class from `index.css` to `<h2>` tags to automatically inherit the global blue decorative underline.

### 3. Adding a New Page
1. Create a new file in `src/pages/NewPage.jsx`.
2. Create its stylesheet in `src/styles/NewPage.css`.
3. Add the route in `src/App.jsx` inside the `<Routes>` block.
4. Wrap the page content in a `<div className="page-container">` or similar wrapper, and use `<section>` tags for semantic HTML.

## 🖥 Available Scripts

In the project directory, you can run:

### `npm install`
Installs all required dependencies. Run this first when cloning the repo.

### `npm run dev`
Runs the app in the development mode utilizing Vite's HMR (Hot Module Replacement).
Open [http://localhost:5173](http://localhost:5173) to view it in your browser.

### `npm run build`
Builds the app for production to the `dist` folder. It correctly bundles React in production mode and optimizes the build for the best performance.

### `npm run preview`
Locally preview the production build generated by `npm run build`.

## 🐳 Production Deployment (Docker)

This project includes a multi-stage `Dockerfile` tailored for cloud deployment (AWS ECS, Google Cloud Run, Azure Container Apps).

1. **Build the Image:**
   ```bash
   docker build -t slc-frontend:latest .
   ```
2. **Run the Container:**
   ```bash
   docker run -p 80:80 slc-frontend:latest
   ```

*Note on Routing:* The Docker image uses an internal `nginx.conf` designed for Single Page Applications. It includes a `try_files $uri /index.html` directive, ensuring that direct hits to sub-routes (like `/careers` or `/services/hr`) do not return 404 errors from Nginx.

## 🖱 Advanced Features to Note
* **Custom Cursor (`CustomCursor.jsx`):** The application overrides the default mouse pointer on desktop devices. If you are adding interactive elements (buttons, links), ensure they use standard semantic HTML (`<a>`, `<button>`) so the cursor automatically expands on hover.
* **Intersection Observers:** Pages like `Careers.jsx` and `About.jsx` use native `IntersectionObserver` APIs to trigger fade-in and scroll-reveal animations. Look for the `reveal-block` CSS class.
